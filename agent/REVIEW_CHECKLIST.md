# Review Checklist

Use this before finalizing a change.

- Does the code still match the migration-assistant posture instead of pretending full conversion?
- Does unsupported content remain recoverable?
- Did shared contract changes update all consumers?
- Were tests added or updated?
- Were docs updated in the same change?
- If CLI behavior changed, do examples and help text still match?
- If artifacts changed, are compatibility implications documented?
- Are any new assumptions stated explicitly?
